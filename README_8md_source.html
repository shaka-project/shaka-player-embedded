1 # Shaka Player Embedded
2 
3 Shaka Player Embedded is a framework that runs [Shaka Player][] in native (C++)
4 apps. This gives a cross-platform interface to Shaka Player allowing native
5 apps to use the player. Your native apps can now be built with the same
6 features, behavior, and API as your Shaka-based web apps.
7 
8 In addition to the C++ API, we provide a high-level wrapper for iOS called
9 ShakaPlayerView which can be used from either Objective-C or Swift.
10 
11 [Shaka Player]: https://www.github.com/google/shaka-player
12

14 ## Platform support
15 
16 We only support iOS at this time, but other platforms could be added.
17 
18 Because many on our team use Linux, there is a Linux port included to make it
19 easier to work on non-iOS-specific features. Linux is not, however, a
20 first-class target platform at this time.

51 ## Setting up for development
52 
53 1. The source is managed by Git at
54 <https://www.github.com/google/shaka-player-embedded>. You will need these
55 tools available on your development system:
56 
57 * Git v1.7.5+
58 * Python v2.7
59  * Requires installing package `enum34`
60 * autoconf 2.57+
61  * m4
62  * automake 1.7+
63  * perl
64 
65 This does not require a C++ compiler as it uses a pre-compiled version of
66 clang downloaded from Google Cloud Storage. However, it does require a C++11
67 compatible C++ library (installed with most compilers, e.g. gcc).
68 
69 2. Install Chromium depot tools, which contains ninja and other required tools.
70 See <https://www.chromium.org/developers/how-tos/install-depot-tools> for
71 details.
72 
73 3. Get the source
74 
75 ```shell
76 git clone https://github.com/google/shaka-player-embedded.git
77 cd shaka-player-embedded
78 ```
79 
80 We use git submodules to manage third-party dependencies. You can run
81 `git submodule update` (or pass `--recurse-submodules` to `git clone`) to
82 download all the submodules; or you can wait and have the `configure` script
83 download only those you need.
84 
85 4. Configure and Build
86 
87 You need to run `configure` to setup configuration settings. You can pass
88 --help to see some of the options you can use. We don't support in-tree builds;
89 you can either run `configure` in another directory or use the --config-name
90 parameter. If you use --config-name, it will create a directory in the source
91 root to hold the data.
92 
93 Once `configure` is done, you can build using `build.py`. There is a Makefile
94 that will run the script for you if you prefer. If you used --config-name,
95 you'll need to use that option in `build.py` (the Makefile will do this for
96 you).
97 
98 ```shell
99 ./configure --config-name Debug
100 make
101 ```
102 
103 Or
104 
105 ```shell
106 mkdir foo
107 cd foo
108 ../configure
109 make
110 ```

113 ## Running the checks
114 
115 If you intend to send a pull-request, you need to ensure the code complies with
116 style guidelines. This is done with the `shaka/tools/presubmit.py` script,
117 which can be run with `make check`.
118 
119 To run all the checks, you also have to have `clang-tidy` installed. It needs
120 to be installed on `PATH`, or you can pass `--clang-tidy` to the script to tell
121 it where to find the binary.
